Task Tracer 是一个面向个人使用的任务管理器，用来管理任务、学习计划、重复事项、本地提醒和长期任务记录。它把项目、标签、日期、子任务、归档和统计放在一个浏览器本地应用中。

它适合学习计划、个人事务、项目跟进、例行复盘，以及任何更重视自主掌控、清晰组织、提醒和状态反馈的任务管理场景。

## 功能地图

| 模块 | 已包含能力 |
| --- | --- |
| 任务生命周期 | 新增、编辑、完成、标记未完成、删除、归档、从归档恢复、清除已完成任务、归档已完成任务。 |
| 日期与截止时间 | 可选本地日期与时间、无日期任务、时区兼容迁移、实时倒计时、进度条和状态颜色。 |
| 状态模型 | 安全、警告、紧急、已过期、已完成、已归档、无截止日期等状态拥有不同标签和视觉处理。 |
| 子任务 | 添加子任务、行内编辑子任务文本、单独完成子任务，并在父任务上展示子任务进度。 |
| 重复任务 | 支持每天、每周、每月和自定义天数周期；完成当前一期后创建下一期，同时保留已完成记录。 |
| 提醒系统 | 提前提醒、重复提醒间隔、稍后提醒、错过提醒补提示、通知权限处理和后台提醒限制说明。 |
| 项目与标签 | 每个任务一个主项目、多个标签、项目视图、标签片段，以及项目/标签参与搜索。 |
| 搜索与筛选 | 搜索任务名称、描述、项目、标签和子任务；支持按项目、标签、状态、截止日期进行限定搜索。 |
| 排序与调整顺序 | 智能排序、新创建、截止日期、名称排序、手动拖拽排序、触摸排序和键盘排序。 |
| 视图体系 | 列表、日历、时间线和统计视图，分别覆盖日常处理、日期排布、时间顺序浏览和效率反馈。 |
| 数据可迁移 | JSON 导出、导入预览、同名冲突处理、替换导入、合并导入、带版本说明的本地备份和备份健康状态。 |
| PWA 能力 | 菜单安装入口、手动安装指引、离线加载、应用图标和新版本刷新提示。 |
| 使用体验 | 明暗主题、响应式布局、减少动态效果支持、状态过渡和中英文界面。 |
| 无障碍 | 键盘导航、清晰焦点、弹窗焦点陷阱、读屏标签、状态播报和自动化 axe 审计。 |

## 核心流程

Task Tracer 从一个简单列表开始，只在需要时逐步加入结构：

1. 添加任务名称、可选描述、截止日期与时间、提醒、重复规则、项目、标签和子任务。
2. 在列表视图中完成日常操作，例如完成、稍后提醒、编辑、归档、删除和手动排序。
3. 当日期排布比列表顺序更重要时，切换到日历视图或时间线视图。
4. 在统计视图中查看完成率、进行中任务逾期率、归档数量、今日完成数和连续完成天数。
5. 在清理浏览器数据、更换设备或测试导入前，查看备份健康状态并下载备份文件。

## 项目、标签与搜索

项目是主要分组。一个任务属于一个项目，例如 `工作`、`个人` 或某个具体产品。项目选择器可以收窄任务列表，全部项目的智能视图会按项目保持分组。

标签是灵活的辅助标记。一个任务可以拥有多个标签，例如 `设计`、`考试`、`复盘`。标签适合描述跨项目的共同特征。

搜索会匹配任务名称、任务描述、项目名称、标签和子任务文本。项目负责组织主要任务空间，标签负责为跨项目检索增加额外入口。

限定搜索可以直接输入到搜索框里，也可以和普通关键词混用：

| 输入 | 匹配内容 |
| --- | --- |
| `project:工作` | 项目名包含“工作”的任务。 |
| `tag:学习` 或 `#学习` | 拥有对应标签的任务。 |
| `status:active` | 进行中的未完成任务。 |
| `status:completed` | 已完成任务。 |
| `status:archived` | 已归档任务，即使当前主筛选是进行中也会显示。 |
| `status:overdue` | 已过期且未完成的任务。 |
| `status:no-deadline` | 无截止日期任务。 |
| `status:repeat` | 重复任务。 |
| `due:today` | 今天截止的任务。 |
| `due:tomorrow` | 明天截止的任务。 |
| `due:week` | 未来七天内截止的任务。 |
| `due:2025-05-20` | 指定本地日期截止的任务。 |
| `due:no-deadline` | 无截止日期任务。 |
| `project:工作 tag:报告` | 同时满足两个限定条件的任务。 |

搜索值以空格分隔。如果项目名或标签名包含空格，建议输入其中有区分度的一部分，例如用 `project:Work` 代替 `project:Work Plan`。

## 视图体系

| 视图 | 用途 |
| --- | --- |
| 列表 | 日常执行任务，包含完整操作、子任务、状态片段、提醒、归档控制和手动排序。 |
| 日历 | 按月份查看日期安排，支持上一月/下一月、上一年/下一年、回到今天、日期详情和无日期分组。 |
| 时间线 | 按日期分组，适合顺着时间检查即将到来或历史阶段的任务负载。 |
| 统计 | 展示完成率、进行中任务逾期率、归档数量、今日完成数、当前连续完成天数和近期完成趋势。 |

## 导入、导出与备份

Task Tracer 把数据操作拆成不同入口，让每个动作的用途更明确。

| 动作 | 用途 |
| --- | --- |
| 导出 | 下载当前任务 JSON，适合迁移、人工检查或手动保存。 |
| 导入 | 读取 JSON 文件，先展示预览，提示导入数量、同名重复项、当前数据影响，再确认替换、合并、保留两份、跳过导入或用导入项替换本地同名任务。 |
| 备份 | 下载带版本信息的快照，包含结构说明、本地日期、提醒时间、重复规则、完成时间和归档状态；菜单中会显示最近备份是否健康。 |

备份会在本地记录最近备份时间。存在任务数据时，Task Tracer 会显示今天已备份、最近已备份，或建议重新备份。

## 安装与更新

浏览器提供安装能力时，可以从应用菜单直接安装 Task Tracer。如果当前浏览器没有自动弹出安装窗口，同一个菜单入口会显示桌面 Chrome/Edge、Android Chrome 和 iOS Safari 的手动安装方式。

当离线应用有新版本准备好时，Task Tracer 会显示更新提示，并提供刷新更新操作，让已安装应用可以明确切换到最新版本。

## 提醒模型

当浏览器支持通知且用户授予权限时，Task Tracer 可以在应用打开期间检查提醒、按设定间隔重复提醒、将提醒稍后处理，并在应用重新运行时补提示错过的提醒。

浏览器通知不是系统级闹钟服务。提醒是否准时投递会受到浏览器策略、操作系统行为、电池设置、标签页生命周期，以及应用是否处于打开或被唤醒状态影响。

## 数据与隐私

任务数据保存在当前浏览器的 IndexedDB 中。Task Tracer 不需要账号，默认不会把任务内容上传到服务器。

更换浏览器、清理站点数据、重新安装应用或迁移设备前，请先使用“立即备份”或“导出”下载 JSON 文件。

## 运行方式

直接访问在线地址：

```text
https://todo.muquew.com/
```

或本地运行：

```bash
git clone https://github.com/muquew/Task-Tracer.git
cd Task-Tracer
python3 -m http.server 8080
```

然后打开：

```text
http://127.0.0.1:8080/
```
